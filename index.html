Minimal Viable Product - toote, teenuse või teenuse testversioon, millel on minimaalne hulk funktsioone (mõnikord isegi üks), mis pakub lõppkasutajale väärtust. MVP luuakse selleks, et testida hüpoteese ja kontrollida kavandatud toote elujõulisust, seda, kui väärtuslik ja nõutud see on turul. Minimaalselt elujõulise toote testimise tulemused ja sihtrühma tagasiside aitavad mõista, kas projekti tuleks edasi arendada, milliseid muudatusi tuleks strateegiasse teha ja mis tuleks jätta algsel kujul. Looge kaug-reposiit

  • Minge GitHubi (või GitLabi, Bitbucketi) ja looge uus reposiit

Algatage kohalik Git reposiit: Oma projekti kaustas käivitage järgmised käsud

git init git add . git commit -m "Algne commit"

Lisage kaug-reposiit: Kopeerige oma kaug-reposiidi URL ja käivitage järgmine käsk

git remote add origin https://github.com/username/repo_name.git

Saadke muudatused kaug-reposiiti

git push -u origin master

Nüüd on teie kohalik reposiit ühendatud kaug-reposiidiga ja muudatusi saab edaspidi saata käsuga git push
